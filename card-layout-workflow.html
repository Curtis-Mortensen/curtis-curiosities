Overview

Board-game card art is usually designed in Photoshop, Figma, or Affinity at a known print resolution. nanDeck then places dynamic content — names, rules text, icons, spreadsheet-driven images — on top of frame artwork using a text script (.nde). The hard part is translating “put the name here on the mockup” into exact coordinates nanDeck understands.

This project standardizes that translation at 600 DPI (high-quality print) while supporting other physical sizes such as poker cards (2.5″×3.5″) and US Letter sheets (8.5″×11″).

Image → Pixel Locations → nanDeck

Design image (PNG/JPG) │ ▼ Region picker — mark rectangles on the art │ outputs: x, y, width, height in pixels ▼ Unit conversion — pixels ÷ DPI → inches (or cm/mm) │ poker @ 600 DPI: 1500 × 2100 px = 2.5 × 3.5 in ▼ nanDeck script — IMAGE / TEXT / RECTANGLE directives │ UNIT=INCH, DPI=600, CARDSIZE=2.5,3.5 ▼ nanDeck build — preview, iterate, export print PDF/PNG

Coordinate convention (both tools)

  • Origin (0, 0) is the top-left of the card.
  • Regions are axis-aligned rectangles: x, y, width, height.
  • nanDeck accepts inches, cm, or mm via UNIT; this repo uses inches.
  • Formula: inches = pixels / DPI

If your mockup was exported at 300 DPI, either re-export at 600 or measure in the picker and enable Scale regions to target card size so coordinates map to the physical card regardless of source resolution.

nanDeck essentials

nanDeck is a Windows scripting tool for generating card decks from a text script. Each line is a directive. Scripts are rerun per card; ranged directives (e.g. 1-18) only apply to matching cards. See the official manual and getting started guide.

Core directives

UNIT = INCH
DPI = 600
CARDSIZE = 2.5, 3.5
PAGE = 8.5, 11, PORTRAIT, HV
LINK = ItemCards.xlsx

IMAGE = 1-10, "frame.png", 0, 0, 2.5, 3.5, 0, N
FONT = Arial, 11, BT, #FFFFFF
TEXT  = 1-10, [Name], 0.2, 0.1, 2.26, 0.22, left, center

Positioning parameters

DirectivePosition paramsNotes
IMAGE x, y, width, height, angle, flags N = no resize; CN = center; CP = center preserve aspect
TEXT x, y, width, height, h-align, v-align wordwrap for body text; FONT line precedes each block
RECTANGLE x, y, width, height, color Useful for debug overlays while tuning layout

Spreadsheet-driven cards

LINK = file.xlsx loads one row per card. Column names in square brackets become fields: [Name], [Game Text]. Nested labels like [label_[Type]] resolve at runtime. Conditional blocks use IF / ENDIF.

Write-up: existing cards.nde

The repo currently contains a single nanDeck script for item cards. It targets standard poker size on US Letter, pulls rows from ItemCards.xlsx, and layers artwork and text in a fixed z-order.

Global setup

UNIT = INCH
CARDSIZE = 2.5, 3.5
PAGE = 8.5, 11, PORTRAIT, HV
LINK = ItemCards.xlsx
[all] = 1-{(Name)}

One card is generated per spreadsheet row that has a Name. The page directive tiles cards in a horizontal-vertical grid on letter paper.

Patterns worth preserving

  • Tag lookup table — [img_bow] = bow.png plus a blank fallback for static validation.
  • Type-based frames — IF = [Type] = Basic selects frame_basic.png, etc.
  • Reusable variables — [tag_x1]…[tag_x4] for icon positions.
  • Conditional flavor text — only rendered when the column is non-empty.
  • Dynamic type label — [label_[Type]] maps type to display string.

When updating layout from a new design mockup, re-measure these zones (or any new zones) in the region picker, then patch the inch values in cards.nde. Add DPI = 600 near the top if you want nanDeck to render at print resolution (default is 300).

Agent job description

An agent working on this project should be able to:

  1. Ingest a design image — accept a dropped PNG/JPG, confirm its pixel dimensions, and determine whether it matches the target card or page size at the configured DPI.
  2. Measure regions — use the region picker (below) or tools/pixels-to-nandeck.py to produce exact pixel rectangles for each UI slot: illustration, title, body text, icons, type strip, etc.
  3. Convert coordinates — translate pixels to inches (or cm) and round sensibly (typically 2–4 decimal places for inches).
  4. Edit nanDeck scripts — update IMAGE and TEXT lines in cards.nde (or new .nde files), preserving existing patterns: IF blocks, lookup tables, FONT lines, and spreadsheet column references.
  5. Validate output — suggest adding temporary RECTANGLE debug overlays, compare nanDeck preview against the source mockup, and iterate until alignment is within ~1–2 px at 600 DPI (~0.0017″).
  6. Support multiple formats — switch CARDSIZE and PAGE when the user targets tarot cards, mini cards, or full letter layouts; rescale all regions proportionally when only the card size changes.
  7. Document changes — leave comments in the .nde file for non-obvious layout decisions and update this HTML if the workflow changes.

CLI helper: tools/pixels-to-nandeck.py

For automated agent runs, export regions as JSON from the picker above, then: 

python tools/pixels-to-nandeck.py regions.json --dpi 600 --unit inch
python tools/pixels-to-nandeck.py regions.json --scale-to-card --card-size poker
python tools/pixels-to-nandeck.py regions.json --json

The script emits a nanDeck header (UNIT, DPI, CARDSIZE, optional PAGE) plus one directive per measured region. Use --scale-to-card when the source image dimensions do not match the target pixel size at the chosen DPI.
