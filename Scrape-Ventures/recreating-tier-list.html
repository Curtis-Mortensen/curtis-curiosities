Recreating the Mobalytics STS2 Card Tier List

Source page: mobalytics.gg/slay-the-spire-2/tier-lists/cards · Saved snapshot: Scrape-Ventures/STS2/ · Plan date: 2026-07-08

Goal

Build a local, searchable HTML tier-list viewer where each card is a real DOM element (name in text, findable with Ctrl+F), organized by character and tier (S/A/B/C/D), with hover-to-expand full card art — matching the Mobalytics layout closely enough to be useful offline.

1. How the original page works

Mobalytics is a React SPA. The card tier list page is not a static HTML table of cards. It is a featured user-generated document (slug cards, document id 46067e61-5426-4b2d-a176-5d40f3d6df33) rendered by their NGF (Next-Gen Framework) document system.

Browser loads mobalytics.gg/slay-the-spire-2/tier-lists/cards │ ▼ React app (chunks in STS2/*.js) │ ▼ POST https://mobalytics.gg/api/sts2/v1/graphql/query query Sts2UgFeaturedDocumentQuery($input: { slug: "cards", type: "tier-lists", ... }) │ ▼ Response contains: document.data.tierLists.values[] ← per-character tier list data └── tierSections[] ← S, A, B, C, D rows └── staticDataItems[] ← cards in that tier ├── name ← searchable text (missing in saved DOM!) ├── slug ← e.g. "crash-landing" ├── iconUrl ← small thumbnail (webp/avif) └── linkUrl ← wiki link Page also embeds YouTube review videos per character (Ironclad, Silent, Regent, etc.)

Tier list widget behavior (from saved JS analysis)

  • Cards use staticDataItems linked to wiki type Sts2Cards.
  • Thumbnails render as square icons (~102×128px in our saved AVIF files) via iconStyle: "square".
  • On hover, a custom tooltip renders the full iconUrl image (larger card art), plus tier/tags metadata.
  • Tier rows are labeled S, A, B, C, D with color-coded section headers.
  • Character sections use anchor IDs like #1-regent-tier-list-2 (widget index + slug).
  • Necrobinder section may include a "TBD" placeholder tier (visible on live page).

4. Proposed architecture for the recreation

HTML element design (fixes Ctrl+F problem)

Each card in the recreated page should be structured like:

<article class="card-entry" data-slug="crash-landing" data-tier="S" data-character="regent">
  <img class="card-thumb" src="assets/thumbs/crash-landing.avif" alt="">
  <span class="card-name">Crash Landing</span>   <!-- Ctrl+F target -->
  <div class="card-hover" role="tooltip">
    <img src="assets/full/crash_landing.webp" alt="Crash Landing">
  </div>
</article>

The visible grid shows only thumbnails (like Mobalytics). The card name sits in the DOM as text (can be visually-hidden or shown under the icon). Hover/focus scales the full card image.

5. Implementation stages

Stage 1 — Scrape tier list data

  • Write scrape/fetch-tier-lists.py (or a browser bookmarklet) to call Mobalytics GraphQL.
  • Parse tierLists.values → normalize to { character, tiers: { S: [...], A: [...], ... } }.
  • Output data/tier-lists.json with card name, slug, tier, character, iconUrl.
  • Blocker: Cloudflare. Options: run scraper locally in user's browser, export HAR from DevTools, or paste cookies into script.

Done when: JSON contains all ~440 card placements across 5 characters × 5 tiers.

Stage 2 — Consolidate card images

  • Copy/rename STS2/{slug}_19D0.avif → assets/thumbs/{slug}.avif.
  • Download full hover images — prefer Mobalytics iconUrl from JSON; fallback to Spire Codex CDN with slug normalization (- → _).
  • Log any cards missing images (slug mismatches, colorless cards, etc.).

Done when: Every card in JSON has thumb + full image paths that resolve.

Stage 3 — Build static HTML viewer

  • Single-page tier-list/index.html with vanilla HTML/CSS/JS (no React, no build step).
  • Character nav tabs or stacked sections matching Mobalytics order.
  • Tier rows (S–D) with flex-wrapped card grid per row.
  • CSS hover: scale full card image above thumbnail (position absolute / popover).
  • Each card includes visible or screen-reader <span class="card-name"> for Ctrl+F.
  • Optional: in-page search box that filters/highlights cards (better UX than raw Ctrl+F).

Done when: Opening index.html locally shows all tiers; hovering shows full card; searching "Crash Landing" finds the card.

Stage 4 — Parity polish (optional)

  • Match Mobalytics dark purple theme (colors from saved CSS: #2c274f background family).
  • Hash anchors per character section (id="1-regent-tier-list-2").
  • Embed YouTube review videos (gather missing 3 video IDs).
  • Show tier methodology blurb (S–D definitions) at top.
  • Handle Necrobinder "TBD" tier row if present in scraped data.

Stage 5 — Refresh workflow

  • Re-run scraper when Mobalytics updates the list (live page shows "Updated on Jun 24, 2026").
  • Diff JSON to see tier changes between versions.
  • Re-generate HTML from template + JSON (don't hand-edit HTML).

7. Immediate next step

Before building HTML: obtain tier-lists.json from Mobalytics. Fastest path for the user:

  1. Open the tier list page in Chrome/Firefox with DevTools → Network.
  2. Filter for graphql or sts2.
  3. Find the Sts2UgFeaturedDocumentQuery response.
  4. Save the JSON response body to Scrape-Ventures/STS2/tier-lists-raw.json.

Once that file exists, implementation can proceed without needing live API access from the agent.
