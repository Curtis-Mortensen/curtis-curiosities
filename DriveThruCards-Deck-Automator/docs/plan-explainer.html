What this project does

DriveThruCards has a partner beta tool: Create US Poker Deck from Image Files Upload. You upload card pictures; their site converts colors (CMYK) and builds a print-ready PDF. Doing that by hand for a big deck is slow — and one deck tops out around 130 cards.

This project helps in two ways:

  1. Organize your art into batches by shared card backs (and the 130 limit).
  2. Automate clicks in a real Microsoft Edge window you can watch, using Playwright.
Login is required The builder URL does not show the upload screen until you are logged in as a partner. The scripts open Edge so you can sign in yourself.

You watch it in Microsoft Edge not headless

“Headless” means a hidden browser. We are not doing that. The automator launches your installed Edge in a normal window.

First time

Run npm run auth:save. Edge opens. Log into DriveThru. Press Enter in the terminal. Your login is kept in a local profile folder (auth/edge-profile/).

Later runs

The same Edge profile opens again, already logged in. You can see every upload, every “Next”, and every long wait while the site works. Optional: --keep-open leaves the window up at the end.

Organizing cards by backs

Many cards share one back design. The organizer groups those together, then packs groups into folders of at most 130 cards.

You write a simple list (manifest)

Each card says: which front image, which back image, how many copies.

{
  "deckName": "My Game",
  "maxCardsPerBatch": 130,
  "cards": [
    { "id": "001", "front": "fronts/001.jpg", "back": "backs/red.jpg", "qty": 1 },
    { "id": "010", "front": "fronts/010.jpg", "back": "backs/blue.jpg", "qty": 2 }
  ]
}

Then npm run organize -- your-manifest.json builds cards/batches/… with renamed 001back.jpg / 001front.jpg files and a pairing sheet called batch.json.

The DriveThru wizard (what Edge will walk through)

1

Upload back images

Drop all backs for this batch onto the upload box.

2

Edit / auto-correct colors ~10 min

Site may offer to fix colors. In testing this took about ten minutes.

3

Upload front images

Same idea as backs, for the card faces.

4

Edit fronts can be slow

Same auto-correct / warning flow as the backs.

5

Assemble the deck

Pair each front with its back (from batch.json) and finish.

6

Render sheet / publish ~10 min each

Rendering cards onto a sheet took ~10 minutes. Publishing (instead of only downloading a PDF) took another ~10 minutes.

Plan for long waits A full batch can easily spend half an hour just on server processing. That is normal for this tool — the automator is built to wait patiently.

How Playwright knows when to continue

Playwright is a remote control for the browser. The important idea: it can watch the page and only move on when something real changes, instead of “sleep 10 minutes and hope.”

What we wait for

Signal Plain meaning
URL change The address bar moved to a new step (for example, upload → edit).
Something appears A button, table, or “success” message shows up that was not there before.
Busy UI goes away Spinners, “please wait”, or processing overlays disappear.
Download starts For PDF download mode, Edge begins saving a file.

Heartbeats (so it does not look frozen)

While waiting up to 20 minutes for a slow job, the terminal prints a line every 30 seconds like: 

…still waiting for auto-correct colors (backs) (3.5 min so far; DTC jobs often take ~10 min)

When the page finally shifts, you see an OK line with how long it took. That logic lives in src/waits.js.

Why not a fixed sleep? Sometimes a job finishes in 2 minutes; sometimes 12. Waiting for the page to change is faster when the site is quick, and safer when it is slow. Fixed sleeps either waste time or give up too early.

Tiny mental model

click "auto-correct"
        ↓
watch for spinner / busy overlay
        ↓
watch until busy is gone AND the edit page looks ready
        ↓
only then click Next

How you run it (local checklist)

  1. Install once: npm install (Playwright will drive your installed Edge — no separate Chromium required for the happy path).
  2. Put art in a manifest; run npm run organize -- path/to/manifest.json.
  3. Log in once: npm run auth:save → Edge opens → sign in → press Enter.
  4. Run a batch while watching Edge: 
    npm run upload:batch -- --batch cards/batches/batch-01-shared-red --start-url "https://tools.drivethrucards.com/builder/deck/images/back/YOUR_ID" --mode download
  5. To publish instead of download: add --mode make (new product) or --mode update.
  6. To inspect at the end: add --keep-open.
