# Story: Homepage Selection Screen

## Story Information
- **Epic:** Landing Pages
- **Story ID:** STORY-1
- **Status:** In Progress
- **Priority:** High
- **Estimated Effort:** Medium

## User Story
As a visitor to Throwing Starfish Studios website, I want to clearly see options to navigate to either the marketing agency or game studio sections, so that I can quickly access the content relevant to my interests.

## Description
The homepage will serve as a selection screen that directs visitors to either the marketing agency section or the game studio section. This is the entry point to the website and needs to clearly communicate the dual nature of Throwing Starfish Studios while maintaining brand consistency.

## Acceptance Criteria
1. Homepage displays the Throwing Starfish Studios logo prominently
2. Two clear navigation options are presented: "Marketing Services" and "Game Studio"
3. Each option has a brief description of what visitors will find in that section
4. Design is responsive and works on mobile, tablet, and desktop devices
5. Page loads in under 3 seconds
6. Shared footer is present with essential company information
7. Basic SEO elements are implemented (meta tags, descriptions, etc.)

## Technical Notes
- Implement using Next.js as specified in the architecture document
- Use Tailwind CSS for styling
- Create reusable components for the logo and navigation options
- Implement analytics to track which option users select

## Design Notes
- Use the company color palette consistently
- Ensure sufficient contrast for accessibility
- Consider a split-screen design approach
- Animation should be subtle and enhance usability

## Dependencies
- Finalized logo and brand assets
- Color palette and typography decisions
- Basic component library setup

## Tasks
1. Set up Next.js project structure
2. Create basic layout component with header and footer
3. Design and implement the selection screen UI
4. Implement responsive behavior
5. Add navigation links to respective sections
6. Implement analytics tracking
7. Optimize for performance
8. Test across devices and browsers
